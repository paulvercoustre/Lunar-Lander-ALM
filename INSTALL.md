    <article class="markdown-body entry-content" itemprop="text"><h3><a id="user-content-install-pre-built-conda-package-its-easy" class="anchor" href="#install-pre-built-conda-package-its-easy" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Install pre-built conda package (it's easy!)</h3>

<hr>

<ol>
<li>Install miniconda3. Download it from
<a href="http://conda.pydata.org/miniconda.html">here</a></li>
<li><p>Create a new environment (named py34 here) and activate it:</p>

<div class="highlight highlight-source-shell"><pre>$ conda create -n py34 python=3.4
$ <span class="pl-c1">source</span> activate py34</pre></div></li>
<li><p>Install Box2D from my channel</p>

<div class="highlight highlight-source-shell"><pre>$ conda install -c https://conda.anaconda.org/kne pybox2d</pre></div>

<p>Recent builds should be available for Windows, Linux, and OS X, with Python
2.7, 3.3, 3.4 and 3.5.</p></li>
<li><p>Alternatively, you can build pybox2d from its source code. You should only
really do this if you want the latest features not available in the
release or if you want to help contribute to pybox2d. See the section below
corresponding to your operating system for more instructions on how to do
this.</p></li>
</ol>

<h3><a id="user-content-the-testbed-examples" class="anchor" href="#the-testbed-examples" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>The testbed examples</h3>

<hr>

<ol>
<li><p>If using the conda packages, install a backend such as pygame to use as a
renderer. I have packages for pygame and pygame_sdl2, the latter of which
works better on OS X currently.</p>

<div class="highlight highlight-source-shell"><pre>$ <span class="pl-c1">source</span> activate py34
$ conda install -c https://conda.anaconda.org/kne pygame_sdl2</pre></div>

<h3><a id="user-content-building-from-source-linux" class="anchor" href="#building-from-source-linux" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building from source: Linux</h3>

<hr>

<ol>
<li><p>Download and install the latest version of SWIG (preferably 3.0+) with 
your package manager.</p>

<p>If you are using Ubuntu, you can install it via Synaptic Package Manager
(package name 'swig'). You will also need to install the python-dev package,
and build-essential. Finally, you'll need python-pygame if you want to run
the testbed.</p>

<div class="highlight highlight-source-shell"><pre>$ sudo apt-get install build-essential python-dev swig python-pygame git</pre></div></li>
<li><p>Clone the git repository</p>

<div class="highlight highlight-source-shell"><pre>$ git clone https://github.com/pybox2d/pybox2d</pre></div></li>
<li><p>Build and install the pybox2d library</p>

<div class="highlight highlight-source-shell"><pre>$ python setup.py build

<span class="pl-c"><span class="pl-c">#</span> Assuming everything goes well...</span>
$ sudo python setup.py install</pre></div></li>
</ol>

<h3><a id="user-content-building-from-source-windows" class="anchor" href="#building-from-source-windows" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building from source: Windows</h3>

<hr>

<ol>
<li>Install MinGW and then MSYS so that you can compile Box2D and pybox2d.
Alternatively, install the correct version of Microsoft Visual Studio for
your version of Python.</li>
<li><p>Install SWIG for making the Python wrapper. Install it in a location in your
PATH, or add the SWIG directory to your PATH.</p></li>
<li><p>Clone the source from the git repository:</p>

<div class="highlight highlight-source-shell"><pre>$ git clone https://github.com/pybox2d/pybox2d</pre></div></li>
<li><p>Run MSYS and locate your pybox2d directory</p>

<div class="highlight highlight-source-shell"><pre>$ <span class="pl-c1">cd</span> /c/path/to/pybox2d</pre></div></li>
<li><p>Build and install pybox2d</p>

<div class="highlight highlight-source-shell"><pre>$ python setup.py build
$ python setup.py install</pre></div></li>
</ol>

<h3><a id="user-content-building-from-source-os-x" class="anchor" href="#building-from-source-os-x" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Building from source: OS X</h3>

<hr>

<ol>
<li>Install homebrew from <a href="http://brew.sh/">http://brew.sh/</a></li>
<li><p>Install SWIG, which wraps the C++ library. From a terminal window:</p>

<div class="highlight highlight-source-shell"><pre>$ brew install swig</pre></div></li>
<li><p>Clone the source from the git repository:</p>

<div class="highlight highlight-source-shell"><pre>$ git clone https://github.com/pybox2d/pybox2d pybox2d_dev
$ <span class="pl-c1">cd</span> pybox2d_dev</pre></div></li>
<li><p>Build and install pybox2d</p>

<div class="highlight highlight-source-shell"><pre>$ python setup.py build

<span class="pl-c"><span class="pl-c">#</span> Assuming you're using either Python from conda or brew Python,</span>
<span class="pl-c"><span class="pl-c">#</span> you should be able to install without superuser privileges:</span>
$ python setup.py install

<span class="pl-c"><span class="pl-c">#</span> For development purposes, it may be more convenient to do a develop</span>
<span class="pl-c"><span class="pl-c">#</span> install instead</span>
$ python setup.py develop</pre></div></li>
</ol>
